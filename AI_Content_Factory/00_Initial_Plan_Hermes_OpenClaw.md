# Контент-завод — пошаговая инструкция (первичный план)

> **Источник:** деление от 10.05.2026, на базе Hermes Agent + OpenClaw stack.
> **Цель:** автоматизированный pipeline от мониторинга AI-новостей до публикации контента (Reels/YouTube/посты) в соцсетях.
> **Связано в Notion:** [Content Factory · AI News](https://www.notion.so/35cc12e061ce81419141f469d4e338d8) • [Deadline master](https://www.notion.so/35cc12e061ce81d1810ccbf32c730e7b)

---

## Архитектура системы

| Уровень | Компонент | Роль |
|---|---|---|
| **Brain** | Hermes Agent | самообучающийся агент с долгосрочной памятью (планирование) |
| **Orchestrator** | OpenClaw | управление инструментами, API и под-агентами (исполнение) |
| **Инфраструктура** | VPS | сервер с поддержкой NPU или достаточным RAM для локальных моделей |

---

## Пошаговая инструкция

### Этап 1: Подготовка инфраструктуры (VPS)

1. **Выбор сервера:** VPS с Ubuntu 24.04/26.04. Минимум **16 ГБ RAM** (лучше 32 ГБ), так как будут запускаться две агентские среды.
2. **Установка Docker:** Самый простой способ развернуть обе системы — использовать **Docker Compose** для изоляции зависимостей.
3. **API ключи:** Подготовить ключи (через **Atlas Cloud** или напрямую) для **Kimi K2.6** или **Claude 4.6** — они показывают лучшие результаты в агентских задачах.

### Этап 2: Установка «Мозга» (Hermes Agent)

1. **Инсталляция:** Установить **Hermes v0.8.0+**. Он будет хранить контекст предпочтений и обучаться на правках.
2. **Настройка памяти:** Подключить векторную базу (**Milvus** или **Pinecone**), чтобы Hermes помнил, какие новости были выбраны ранее.
3. **Интеграция:** Настроить **Telegram-бота** как интерфейс взаимодействия с Hermes. Дайджесты приходят туда.

### Этап 3: Установка «Оркестратора» (OpenClaw)

1. **Развертывание:** Установить OpenClaw для управления рутинными задачами.
2. **Подключение навыков (Skills):**
   - **Web Search Skill** — для поиска новостей
   - **Last 30 Days Skill** — для фильтрации только свежего контента
   - **Social Media Skills** — коннекторы для Telegram, Instagram, WordPress

## Рабочий процесс (Workflow)

### Шаг 1: Ежедневный мониторинг и Дайджест

- OpenClaw по расписанию (**cron job в 8:00 утра**) запускает поиск по заданным ключевым словам (`AI agents trends 2026`, `LLM benchmarks`).
- Собранные данные передаются в **Hermes Agent**.
- Hermes анализирует, убирает шум и формирует структурированный дайджест.
- **Результат:** в Telegram сообщение «Сегодня 5 важных новостей. Какую разберём подробно?»

### Шаг 2: Выбор и Сценарная работа

- Ответ Hermes: «Номер 3 про Kimi K2.6».
- Hermes даёт команду OpenClaw произвести **Deep Research** по конкретной теме.
- Получив фактуру, Hermes активирует роль **Scriptwriter**. Используя стиль (из памяти), пишет сценарий для видео (Reels или YouTube).
- Сценарий включает: **Hook (зацепка)** → **основная часть** → **Call to Action**.

### Шаг 3: Производство контента (Медиа-слой)

Через сторонние сервисы по API (настройка в OpenClaw):

1. **Генерация аватара:** сценарий → **HeyGen** или **Kling AI** для видео с цифровым клоном.
2. **Генерация визуала:** для каруселей OpenClaw использует **Recraft** или **Flux** для стильных картинок.
3. **Сборка:** вся информация (текст + видео/фото) собирается в финальный пост.

### Шаг 4: Постинг и Аналитика

- OpenClaw отправляет готовый контент в **черновики** соцсетей (или публикует сразу после одобрения в Telegram).
- После публикации Hermes отслеживает реакцию (лайки/комменты) и сохраняет данные → следующий раз предлагает новости, которые лучше залетают у аудитории.

## Рекомендации к добавлению

1. **Фильтр «Human-Like»** — внедрить инструкцию с **27 фильтрами** (как в гайде Георгия Риверы), чтобы тексты не выглядели роботами.
2. **Валидатор** — паттерн **Review Chain**: один агент пишет сценарий, второй (в режиме критика) проверяет на ошибки и логику перед показом.
3. **Локальный хостинг** — если бюджет на API будет расти (Cost Shock), рассмотреть перенос части задач на локальные модели через **vLLM** на **AMD Developer Cloud** (бесплатно или очень дешево).
