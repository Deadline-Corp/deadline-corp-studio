# 🤖 БОТ — Deadline Sales Manager

Предполагаемый функционал автономной системы продаж для агентства Deadline. Заменяет менеджера по продажам полностью. Работает 24/7 по всем каналам входящего трафика.

---

## КРАТКАЯ ВЕРСИЯ

### Что это

Система, которая полностью заменяет менеджера по продажам. Работает 24/7, не теряет ни одного лида, ведёт клиента от первого сообщения до оплаты и дальше — к повторным продажам. Администратор управляет через понятный интерфейс без программирования. Продукт: IT-решения для бизнеса.

---

### Привлечение и квалификация

Принимает входящие из всех каналов: Telegram, Instagram, WhatsApp, TikTok, сайт, email. Все обращения стекаются в единую базу — один человек из разных каналов склеивается в один профиль. Каждому контакту присваивается балл приоритета: реклама и прямые запросы — в первую очередь, реакции и комментарии — в последнюю. Определяет, насколько клиент готов к покупке — от холодного до готового.

### Ведение диалога и продажа

Управляет разговором через вопросы, не давит, держит темп. Знает продукт, конкурентов и боли клиентов. Понимает, что клиент смотрит несколько вариантов, и отрабатывает это с первых минут. Если клиент торгуется — может предложить скидку или бонус, но только когда видит реальный риск потери: не всем подряд. При появлении акции — сам рассылает предложение подходящим клиентам.

### Созвон и формирование ТЗ

Записывает и транскрибирует созвон. Формирует резюме и отправляет клиенту интерактивный список пунктов на подтверждение — каждый пункт отдельно. После согласования — генерирует коммерческое предложение и отправляет на утверждение администратору. Работа начинается только после двойного подтверждения.

### Работа с паузами и молчанием

Если клиент прервал диалог — возвращается через настраиваемое время с учётом причины паузы. Если молчит долго — греет постепенно: кейсы, новости, акции, поздравления. Интенсивность зависит от «температуры» лида. Если клиент явно отказал — фиксирует с причиной и возвращается через несколько месяцев.

### После сделки

Информирует клиента о ходе работ по расписанию. Контролирует даты платежей и мягко напоминает о них. После завершения проекта собирает отзыв и предлагает агентскую программу — комиссию за приведённого клиента. Предлагает следующую услугу в нужный момент, не навязчиво.

### Для администратора

Все диалоги в одном окне. Карточка подготовки к созвону. Еженедельная аналитика по воронке. Можно вмешаться в любой диалог в любой момент. Настройки — через текст или голос, воронку можно отредактировать вручную перед применением. Три режима: полный автопилот, совместная работа, ручное управление.

### Суть системы

Операционная система агентства, где AI — главный исполнительный слой. Выручка становится функцией от качества системы, а не от количества сейлзов. Масштабирование без линейного найма. Конверсия предсказуемая. Воронка дисциплинированная.

---

### 1. Архитектура — 4 слоя ✅

**Слой 1 — Приём и нормализация:** входящие из всех каналов → единый формат → идентификация → устранение дублей.

**Слой 2 — Мозг:** языковая модель + база знаний + память о каждом клиенте.

**Слой 3 — Оркестрация:** машина состояний воронки, скоринг, расписание касаний, полномочия на скидки, передача управления.

**Слой 4 — Интерфейс администратора:** единое окно, диалоги, задачник, настройки, аналитика.

---

### 2. Каналы трафика ✅

Модульная архитектура. Telegram, Instagram, WhatsApp, TikTok, сайт (чат и форма), email. Каналы подключаются независимо. Голосовые звонки в реальном времени — возможно в будущем, сейчас не прорабатывается.

---

### 3. Идентификация контакта ✅

Один человек из разных каналов → один профиль. Склейка по email или телефону при первом контакте.

---

### 4. Типы входящих взаимодействий ✅

- **P1** — Прямое сообщение с запросом / клик по рекламе → немедленная воронка
- **P2** — Форма с сайта / прямое сообщение без потребности
- **P3** — Возврат холодного лида / позитивный ответ
- **P4** — Нейтральные ответы, комментарии
- **P5** — Stories, реакции
- **P6** — Задачник (написать первым)
- **Hard Stop** — явный отказ → автокасания прекращаются, профиль сохраняется с причиной, реактивация через настраиваемый срок

---

### 5. Квалификация и приоритизация ✅ / 🔲

**Система весов:** базовый балл (тип взаимодействия) + балл за содержание (ключевые слова) + балл за источник. Порог настраивается в админке.

**Decay:** не ответил 48ч → приоритет снижается на ступень автоматически.

**BANT-квалификация 🔲:** в первых 2-3 вопросах ненавязчиво: бюджет, ЛПР, потребность, сроки.

**Адаптивный скоринг 🔲:** после накопления данных — пересчёт весов по реальным конверсиям.

---

### 6. Три модуля знаний

**Продажная логика ✅**

Ведёт по воронке через вопросы. Уровни напористости 1–5 (задаются в админке). Детектор сравнения — замечает «смотрю варианты» и спрашивает о критериях выбора. Детектор «когда отпустить» — 2 негативных сигнала подряд → снижение интенсивности.

**Знание продукта ✅ / 🔲**

Продукт: IT-решения (автоматизация, интеграции, сайты, боты). Боли клиентов: срыв сроков, языковой барьер, непонятно что будет на выходе, страх потерять деньги. Возражения — деревом: возражение → ответ → что делать если не принято → когда прекратить. База конкурентов обновляется регулярно. Конкретное содержание — заполняется после ресёрча 🔲

**Знание клиента ✅ / 🔲**

Профиль: история, интересы, стиль, лояльность, паттерн активности, важные даты. Три уровня сбора: пассивный (из диалогов), контекстный (один вопрос в нужный момент), внешний (публичные данные по ссылке).

---

### 7. Температура лида ✅ / 🔲

| Градация | Сигналы | Стратегия |
| --- | --- | --- |
| Холодный | Случайный контакт | Пассивная фиксация |
| Тёплый | 2+ содержательных ответа | Лёгкое вовлечение |
| Горячий | Спросил цену/сроки/портфолио | Активная воронка |
| Готовый | «Что нужно чтобы начать?» | Предложение, созвон |
| Клиент | Предоплата / договор | LTV-сценарий |

Decay: 14 дней без ответа → -1 ступень. 21+ день → «Замёрзший» → реактивация с новым углом.

---

### 8. Воронки и сценарии ✅ / 🔲

Настройка через промпт или голос. Бот показывает схему → администратор редактирует вручную → подтверждает.

**Два сценария:** сложный запрос → созвон / простой запрос → сбор данных → ТЗ.

**Fallback:** клиент замолчал на 3-4-м вопросе → автопредложение созвона.

Детали воронки и критерий «сложный/простой» 🔲

---

### 9. Коммерческое предложение ✅ / 🔲

После утверждения ТЗ система генерирует КП по шаблону: состав, сроки, стоимость, условия оплаты. Отправляется клиенту в диалог. Администратор может отредактировать до отправки или настроить автоотправку. Шаблон КП 🔲

---

### 10. Скидки, торг и акции ✅ / 🔲

**Полномочия на скидку:** администратор задаёт правила — при каких условиях бот может давать скидку или бонус. Только при реальном риске потери клиента, не всем подряд. Максимальный размер и условия — в админке.

**Сценарий торга:** бот сначала защищает цену через ценность → если не помогает и риск высокий → допустимый бонус/скидка → если бюджет клиента выше порога → передача администратору.

**Акции:** администратор добавляет акцию → система определяет подходящих клиентов → персонализированная рассылка. Логика определения аудитории 🔲

---

### 11. После созвона — ключевая фича ✅ / 🔲

Запись → транскрипция → резюме по категориям → **интерактивная валидация ТЗ** (клиент подтверждает каждый пункт отдельно) → финальное ТЗ администратору → КП → двойное подтверждение → старт.

**Follow-up:** в течение 30 минут после созвона — автоматическое резюме договорённостей клиенту.

**Для администратора:** резюме с отметками времени в записи — можно перейти к спорному моменту.

Этой фичи нет ни у одного конкурента. Закрывает главную боль IT-клиентов — разрыв между заказчиком и исполнителем. Ветка формирования ТЗ 🔲

---

### 12. Голосовые сообщения — 2 модуля ✅ / 🔲

**Модуль А — Голос в диалоге (асинхронный):** транскрипция + метки (сомневался, оборвал мысль). Метки влияют на следующий ответ.

**Модуль Б — Голосовой звонок:** возможно в будущем.

---

### 13. Незавершённые диалоги ✅ / 🔲

**6 типов прерывания и стратегия возврата:**

1. Пропал без объяснений → касание через 2-4ч, возврат к месту остановки
2. «Занят / позже» → пауза 24ч, потом новый угол (не повтор)
3. Назвал дату → пишет именно в этот день
4. Ждёт информации от нас → 24ч «Получили?», 3 дня → другой угол
5. Не ответил на вопрос → 4-6ч, переформулировка проще
6. Поставлен на паузу администратором → ждёт снятия

**Логика:** 1-я попытка (тот же канал) → 2-я (новый угол) → 3-я (смена канала) → прогрев молчащих. Количество попыток и интервалы — в админке.

---

### 14. Прогрев молчащих лидов ✅ / 🔲

| Статус | Срок молчания | Частота | Формат |
| --- | --- | --- | --- |
| Горячий | < 14 дней | Раз в 1-2 дня | Прямой follow-up |
| Тёплый | 15-60 дней | Раз в неделю | Кейс, контент |
| Холодный | 61-180 дней | Раз в 3 нед | Контент + оффер |
| Замёрзший | > 180 дней | Раз в квартал | Спецоффер или архив |

Всё настраивается в админке через промпт. Библиотека сценариев 🔲

---

### 15. Информирование о ходе работ ✅ / 🔲

По расписанию: что сделано, что следующее, всё ли в срок. Администратор отмечает этапы → система автоотправляет клиенту. Формат сообщений 🔲

---

### 16. Апсейл и кросс-продажи ✅ / 🔲

После завершения анализирует профиль и предлагает следующую логичную услугу в нужный момент. Не рандомно — только то, что является естественным следующим шагом. Матрица апсейл-предложений 🔲

---

### 17. Отзывы, рекомендации и агентская программа ✅ / 🔲

После завершения → мягкий сбор отзыва в контексте диалога. При высокой лояльности → предложение агентской программы: комиссия за приведённого клиента. Условия — в админке. Источник нового лида фиксируется автоматически. Механизм реферальной цепочки 🔲

---

### 18. Подготовка к созвону ✅

Автоматическая карточка для администратора перед звонком: кто клиент, история диалога, что важно, что выяснить, на что обратить внимание.

---

### 19. Контроль дебиторки ✅ / 🔲

Отслеживает даты платежей. За N дней — мягкое напоминание клиенту. Просрочка → эскалация администратору. Тон напоминаний — человеческий. Логика тона по степени просрочки 🔲

---

### 20. CRM и этапы воронки ✅ / 🔲

**Этапы:** Новый лид → В диалоге → Квалифицирован → NDA → На созвоне → ТЗ утверждено → КП → Предоплата → В работе → Завершён успешно → Постпродажа

**Завершён неуспешно расщепляется:** по цене / не наш формат / конкурент / отложил / нет бюджета

Конкретная CRM-система — определяется позже 🔲

---

### 21. Передача управления ✅ / 🔲

Администратор перехватывает любой диалог в любой момент.

**7 триггеров уведомления:** низкая уверенность бота / 2 негатива подряд / крупная сделка / юридические слова / петля в диалоге / «соедини с менеджером» / возврат после 24ч молчания.

---

### 22. Режимы работы ✅

- **Автономный** — бот работает сам
- **Co-pilot** — бот предлагает ответ, администратор одобряет (timeout 5 мин → автоотправка)
- **Совместный** — бот ведёт, администратор наблюдает и может вмешаться
- **Ручной** — администратор ведёт сам, бот учится

Confidence gate: низкая уверенность → автоперевод в co-pilot.

- Алерты при триггерах

Визуальная схема воронки 🔲

### 24. Аналитика ✅ / 🔲

Еженедельный дайджест: новые лиды, конверсия по этапам, закрыто успешно/неуспешно, причины потерь, handoff rate.

**Ключевые метрики:** время первого ответа (цель < 5 мин для P1), конверсия по переходам, handoff rate (цель → 10-15%), approval rate ТЗ, причины потерь, стоимость диалога. Дашборды 🔲

### 25. Стабильность и память ✅

24/7, облачный сервер, полная история каждого клиента, восстановление после перезапуска, не путает клиентов, очистка устаревших диалогов по настраиваемым правилам.
