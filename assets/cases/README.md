# Кейсы — как добавлять и заменять ассеты

Вся страница `cases.html` рендерится из одного файла **`/cases-data.js`**.
Чтобы добавить кейс или подставить реальный ассет — правится только этот файл.
Вёрстку (`cases.html`, `cases.css`, `cases.js`) трогать не нужно.

## Где лежат файлы
```
assets/cases/<id>/        ← папка под каждый кейс (id = поле id в cases-data.js)
  poster.jpg              ← статичный кадр (постер видео / превью)
  demo.mp4                ← короткий скринкаст 5–15 c, без звука
  1.jpg, 2.jpg …          ← скриншоты
```
Пример: кейс с `id: 'vrp'` → файлы в `assets/cases/vrp/`.

## Режимы показа (`media.mode`)

| mode          | Что показывает | Какие поля заполнить |
|---------------|----------------|----------------------|
| `placeholder` | Фирменную заглушку (по умолчанию) | ничего |
| `video`       | Видео в карточке, играет при наведении | `video`, `poster` |
| `image`       | Скриншот | `images: ['assets/cases/<id>/1.jpg']` |
| `scroll`      | **Рекомендуется для сайтов/приложений.** Скриншот сам прокручивается в рамке (лёгкая анимация, без live-загрузки) | `image: 'assets/cases/<id>/screen.png'` + `frame: 'browser'`/`'phone'` |
| `iframe`      | Живой сайт прокручивается в рамке (тяжелее — грузит реальную страницу) | `liveUrl: 'https://...'` |
| `chat`        | Анимированный диалог с ботом (печатается) | `chat: [ {from,text:{ru,en}} ]` |

> **Адрес скрыт.** В рамке браузера URL не показывается (замок + точки) — реальная ссылка наружу не видна ни в `scroll`, ни в `iframe`.
> Для `scroll` делай **длинный вертикальный скриншот** страницы целиком (full-page) — он плавно проедет в окне.

> Для `type: 'data'` дополнительно рисуется flow-схема из поля `flow`.

## Заменить плейсхолдер на реальный ассет — 3 шага

1. Положи файл в `assets/cases/<id>/`.
2. Открой `cases-data.js`, найди нужный кейс, поменяй `media`:
   ```js
   // было
   media: { mode: 'placeholder' }
   // стало — видео при наведении
   media: { mode: 'video', video: 'assets/cases/vrp/demo.mp4', poster: 'assets/cases/vrp/poster.jpg' }
   ```
   или живой сайт в рамке:
   ```js
   media: { mode: 'iframe', liveUrl: 'https://viprentalphuket.com' }
   ```
3. Сохрани, обнови страницу. Готово.

## Добавить новый кейс
Скопируй любой объект в массиве `window.CASES`, поменяй `id`, тексты, `type`
(`web` / `app` / `bot` / `data`), `featured` (true = крупная карточка) и `media`.
Фильтр и сетка подхватят его автоматически.

## Рекомендации по ассетам
- Видео: `.mp4` (H.264), без звука, 5–15 c, ≤ 1080p, ≤ 3–4 МБ (lazy-load уже включён).
- Постер: `.jpg`/`.webp`, тот же кадр что первый кадр видео.
- Скриншоты: `.webp` предпочтительно.
- Для бота лучше всего — `mode: 'chat'` со сценарием: выглядит живо и весит 0 КБ.
